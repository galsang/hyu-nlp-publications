Revisiting the Impact of Pursuing Modularity for Code Generation

EMNLP 2024 Findings
Deokyeong Kang, Ki Jung Seo, Taeuk Kim

한줄 요약

모듈형 코드 생성이 LLM 성능을 항상 향상시킨다는 통념에 도전하며, 과제 난이도와 모델 능력에 따라 그 효과가 달라짐을 실증적으로 밝힌 연구입니다.

배경 및 동기

모듈형 프로그래밍 — 문제를 작은 재사용 가능한 하위 함수로 분해하는 방식 — 은 소프트웨어 공학의 핵심 원칙입니다. 최근 LLM 기반 코드 생성 연구에서도 이 원칙을 적용하여, 모델이 최종 솔루션을 작성하기 전에 문제를 하위 함수로 분해하도록 유도하는 접근법이 등장했습니다.

가정: 모듈형 프롬프팅 전략(예: 하위 함수를 먼저 생성)이 좋은 소프트웨어 공학 관행처럼 코드 생성 품질을 향상시킬 것이라는 통념이 있었습니다.

핵심 질문: 모듈화 추구가 다양한 모델, 벤치마크, 문제 난이도에서 실제로 더 나은 코드 생성으로 이어지는가?

방법론

저자들은 모듈형 vs 비모듈형 코드 생성 전략을 여러 축에서 체계적으로 평가합니다:

  • 다양한 LLM: 오픈소스 및 상용 모델을 다양한 크기로 실험하여 일반화 가능성 검증
  • 다양한 벤치마크: HumanEval, MBPP, APPS, CodeContests 등 쉬운 문제부터 대회 수준까지 포괄하는 벤치마크에서 평가
  • 모듈화 전략: 계획 후 코딩, 하위 함수 분해, 단일 함수 생성 등 다양한 모듈형 프롬프팅 방식 비교
  • 세분화 분석: 문제 난이도, 코드 복잡도, 모델 능력별로 결과를 세분화하여 모듈화가 도움이 되는 경우와 그렇지 않은 경우를 식별

주요 결과

  • 보편적 이점 없음: 모듈형 생성이 단일 함수 생성보다 일관되게 우수하지 않으며, 많은 설정에서 동등하거나 오히려 낮은 성능을 보임
  • 난이도의 영향: 모듈화는 어려운 대회 수준 문제에서는 도움이 되지만, 단순한 문제에서는 불필요한 복잡성을 유발하여 오히려 성능 저하
  • 모델 능력의 영향: 강력한 모델은 모듈형 전략의 혜택을 더 받지만, 약한 모델은 분해의 추가 부담으로 어려움을 겪음
  • 오류 전파: 모듈형 접근법은 연쇄 오류에 취약 — 하위 함수가 잘못되면 전체 솔루션이 실패

왜 중요한가

이 연구는 LLM 기반 코드 생성에서 "모듈화가 항상 좋다"는 기존 가정에 도전합니다. 세밀한 실증 분석을 통해, 연구자와 실무자가 모듈형 프롬프팅 전략을 언제 적용하고 언제 단순한 단일 생성이 더 나은 선택인지 판단할 수 있는 근거를 제공합니다. 최적의 전략은 문제 난이도와 모델 능력 간의 상호작용에 달려 있으며, 하나의 접근법이 모든 상황에 적합하지 않음을 시사합니다.
