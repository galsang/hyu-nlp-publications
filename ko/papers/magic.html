MAGIC: A Multi-Hop and Graph-Based Benchmark for Inter-Context Conflicts in Retrieval-Augmented Generation

EMNLP 2025 Findings
Jungyeon Lee, Kangmin Lee, Taeuk Kim

한줄 요약

RAG 시스템에서 검색된 문서 간 지식 충돌을 지식 그래프를 활용하여 체계적으로 생성하고, 멀티홉 추론 환경에서 충돌 탐지 및 위치 파악 능력을 평가하는 벤치마크입니다.

배경 및 동기

검색 증강 생성(RAG)은 외부 문서를 검색하여 LLM 응답을 강화하는 기법입니다. 그런데 검색된 문서들이 서로 모순되거나 모델의 내부 지식과 충돌하면 어떻게 될까요?

기존 벤치마크의 한계: 기존 데이터셋(ECON, WikiContradict)은 78%가 단일홉 충돌로 구성되어 단순한 모순만 다룹니다. 실제 환경에서는 여러 문서를 연결해야만 발견할 수 있는 복잡한 모순이 훨씬 더 흔합니다.

제안 방법: KG 기반 충돌 생성 프레임워크

1
서브그래프 추출
Wikidata5M의 825개 관계를 46개로 필터링하고, DFS 순회를 통해 서브그래프 추출
2
충돌 생성
서브그래프 수준의 few-shot 프롬프팅으로 o3-mini를 활용하여 다양한 충돌 생성, 인간 검증 포함
3
텍스트 변환
GPT-4o-mini로 자연어 변환; Claude 3.5 Sonnet으로 트리플릿 커버리지 검증 (정확도: 95.21%)

실험 결과

MAGIC은 1,080개 인스턴스로 구성되며 두 가지 능력을 평가합니다: 충돌 식별(ID)과 충돌 위치 파악(LOC).

벤치마크ID 점수 (%)LOC 점수 (%)
ECON (기존)74.7357.09
WikiContradict (기존)69.9355.74
MAGIC (제안)64.5440.51
  • 최고 성능: GPT-4o-mini가 ID 83.61%, LOC 55%로 최고 성능을 보이지만, 인간 성능(ID 92.5%, LOC 83.3%)에는 여전히 크게 못 미침
  • 멀티홉 vs 단일홉: 멀티홉 충돌에서 평균 ID 점수가 70.91%에서 46.13%로 급격히 하락
  • 위치 파악의 어려움: 충돌의 정확한 위치를 찾는 것이 존재 여부를 탐지하는 것보다 훨씬 어려움 (LOC << ID)
  • 맥락 효과: 문서 길이가 증가할수록 성능 저하

왜 중요한가?

RAG는 현재 가장 널리 사용되는 LLM 배포 패턴입니다. 검색 결과의 모순이 탐지되지 않으면 잘못된 정보가 사용자에게 전달될 수 있습니다. MAGIC은 지식 그래프의 해석 가능한 구조를 활용하여 충돌의 원인과 결과를 추적하고, 실제 환경에서 흔한 멀티홉 충돌을 최초로 체계적으로 평가합니다.
