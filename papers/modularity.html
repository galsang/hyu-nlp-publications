Revisiting the Impact of Pursuing Modularity for Code Generation

EMNLP 2024 Findings
Deokyeong Kang, Ki Jung Seo, Taeuk Kim

One-Line Summary

A comprehensive study revealing that modular code generation — widely assumed to help LLMs — does not consistently improve performance and can even hurt it, depending on the task and model.

Background & Motivation

Modular programming — breaking a problem into smaller, reusable sub-functions — is a cornerstone of software engineering. Recent work on LLM-based code generation has adopted this principle, prompting models to decompose problems into helper functions before writing the final solution.

Assumption: Modular prompting strategies (e.g., generating sub-functions first) should improve code generation quality, mirroring good human engineering practice.

Question: Does pursuing modularity actually lead to better code generation across different models, benchmarks, and problem difficulties?

Method

The authors systematically evaluate modular vs. non-modular code generation strategies across multiple axes:

  • Multiple LLMs: Experiments span both open-source and proprietary models of varying sizes to test generalizability
  • Diverse Benchmarks: Evaluation on established code generation benchmarks (HumanEval, MBPP, APPS, CodeContests) covering easy to competition-level problems
  • Modularity Strategies: Comparison of different modular prompting approaches — including planning-then-coding, sub-function decomposition, and self-contained monolithic generation
  • Fine-grained Analysis: Results broken down by problem difficulty, code complexity, and model capability to identify when modularity helps or hurts

Key Results

  • No universal benefit: Modular generation does not consistently outperform monolithic generation; in many settings it performs comparably or worse
  • Difficulty matters: Modularity tends to help on harder, competition-level problems but can hurt on simpler tasks where decomposition introduces unnecessary complexity
  • Model capability matters: Stronger models benefit more from modular strategies, while weaker models often struggle with the added overhead of decomposition
  • Error propagation: Modular approaches are susceptible to cascading errors — if a sub-function is incorrect, the entire solution fails

Why It Matters

This work challenges the prevailing assumption that "modular is always better" in LLM-based code generation. By providing a nuanced, empirically grounded analysis, it helps researchers and practitioners make informed decisions about when to apply modular prompting strategies — and when simpler, monolithic generation is the better choice. The findings suggest that the optimal strategy depends on the interplay between problem difficulty and model capability, rather than a one-size-fits-all approach.
