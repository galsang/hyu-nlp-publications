Hybrid-Vector Retrieval for Visually Rich Documents: Combining Single-Vector Efficiency and Multi-Vector Accuracy

ACL 2026 Findings
Juyeon Kim, Geon Lee, Dongwon Choi, Taeuk Kim, Kijung Shin

One-Line Summary

A two-stage retrieval framework (HEAVEN) that achieves 99.87% of multi-vector accuracy while reducing computation by 99.82% for visually rich document retrieval.

Background & Motivation

Visually rich documents (PDFs with tables, charts, layouts) require retrieval methods that understand visual structure. Single-vector methods are efficient but inaccurate; multi-vector methods are accurate but computationally expensive. HEAVEN bridges this gap by combining the best of both worlds.

Core Challenge: How can we achieve multi-vector-level accuracy at single-vector-level cost for retrieving visually rich documents at production scale?

Proposed Method: HEAVEN Framework

HEAVEN is a two-stage hybrid retrieval framework that combines single-vector efficiency for initial candidate retrieval with multi-vector precision for reranking, while introducing novel techniques to minimize computation at each stage.

1
Fast Candidate Retrieval
Single-vector search over Visually-Summarized Pages (VS-Pages) that assemble representative visual layouts for efficient initial filtering
2
Precise Reranking
Multi-vector reranking with query tokens filtered by linguistic importance to reduce redundant computation

ViMDOC Benchmark

We introduce ViMDOC, the first benchmark for visually rich, multi-document, long-document retrieval, addressing a critical gap in existing evaluation resources.

  • 76,347 pages across 1,379 documents
  • 10,904 queries with diverse question types
  • 55.4 pages average document length

Experimental Results

  • 99.87% Recall@1 preservation compared to full multi-vector retrieval
  • 99.82% FLOPs reduction per query
  • Consistent performance across multiple benchmarks and document types

Why It Matters

As enterprises deal with millions of PDF documents, efficient yet accurate retrieval is critical. HEAVEN makes production-scale deployment practical by dramatically reducing computational cost while maintaining quality, enabling visually-aware document search at scale without sacrificing the accuracy that multi-vector methods provide.
