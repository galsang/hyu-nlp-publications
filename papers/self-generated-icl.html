Self-Generated In-Context Learning: Leveraging Auto-regressive Language Models as a Demonstration Generator

Workshop on Large-scale Pre-trained Language Models (LPLM 2022) at NAACL 2022
Hyuhng Joon Kim, Hyunsoo Cho, Junyeob Kim, Taeuk Kim, Kang Min Yoo, Sang-goo Lee

One-Line Summary

A method that enables large language models to generate their own in-context demonstrations, eliminating dependence on external training data while significantly outperforming zero-shot learning across text classification tasks.

Background & Motivation

In-context learning (ICL) allows pre-trained language models to solve tasks by conditioning on a few input-label demonstration pairs without any parameter updates. However, ICL has a critical limitation: it heavily depends on the quality and choice of demonstrations, which are typically selected from external datasets.

Key Questions: (1) Can we eliminate the dependency on training datasets by generating demonstrations? (2) If so, how can we create demonstrations with high input-demonstration correlation?

Proposed Method: SG-ICL

Self-Generated In-Context Learning (SG-ICL) leverages the generative abilities of PLMs to create their own demonstrations through two steps:

1
Self-Generation Step
For each test input and each class label, the PLM generates in-context samples using a manually designed generation template conditioned on both the input and the class. This produces demonstrations highly correlated with the test instance.
2
Inference Step
The generated samples are used as demonstrations for standard in-context learning. The PLM performs the downstream task using these self-generated demonstrations concatenated with the test instance.

Experimental Results

Experiments are conducted on four text classification datasets (SST-2, SST-5, RTE, CB) using GPT-J (6B), with 8 self-generated in-context samples per test input.

  • Consistent Improvement: SG-ICL significantly outperforms zero-shot learning across all four tasks, despite having no access to training data
  • Low Variance: Generated demonstrations show much more stable performance with low variance compared to randomly selected demonstrations from training data
  • Worth ~0.6 Gold Samples: Each self-generated in-context sample is generally worth approximately 0.6 gold training samples
  • Outperforms Few-shot: Using 8 self-generated samples outperforms few-shot learning with up to 5 gold training samples
  • Input Conditioning Matters: Conditioning generation on both the input and class label (rather than class alone) produces demonstrations with higher semantic similarity to the test input, leading to better performance

Why It Matters

SG-ICL opens a new paradigm where language models can bootstrap their own demonstrations without any external data. This is particularly valuable in scenarios where labeled data is scarce or unavailable, and it provides a more stable alternative to the notoriously variance-prone random demonstration selection in standard ICL.
